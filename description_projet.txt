DESCRIPTION DU PROJET DE FIN D'ÉTUDES

TITRE: GesClient - Système de Gestion Commerciale Unifiée

--------------------------------------------------------------------------------

PROBLÉMATIQUE

Actuellement, les données des clients sont dispersées dans plusieurs fichiers 
Excel distincts: un pour les clients des ventes et un autre pour les clients 
des collectes. Cette situation crée des doublons, des difficultés de suivi 
et une perte de temps.

OBJECTIF

Créer une base de données unique qui centralise tous les clients (ventes + 
collectes) tout en permettant à chaque acteur de l'entreprise d'accéder 
uniquement aux fonctionnalités qui le concernent.

LES 3 ACTEURS

1. LE DIRECTEUR
   - Accès complet à toutes les fonctionnalités
   - Consultation des statistiques globales
   - Possibilité de supprimer des données
   - Voir l'historique des actions (journal d'audit)

2. LE COMMERCIAL
   - Gère les clients: création, modification
   - Crée et suit les commandes de vente
   - Génère les factures (proforma et définitives)
   - Envoie les factures par WhatsApp aux clients
   - Suit les paiements

3. LE COLLECTEUR
   - Gère les collectes (entrées de stocks)
   - Enregistre les quantités collectées
   - Associe les collectes aux clients

FONCTIONNALITÉS PRINCIPALES

- Gestion centralisée des clients
- Suivi des commandes et paiements
- Génération automatique de factures PDF
- Envoi de factures par WhatsApp
- Import/Export des clients en Excel
- Journal d'audit des actions
- Notifications

TECHNOLOGIES

Backend: NestJS (TypeScript)
Base de données: PostgreSQL
Authentification: JWT

BÉNÉFICES

- Base de données unifiée et centralisée
- Meilleure visibilité sur l'ensemble des clients
- Gain de temps dans la gestion quotidienne
- Communication plus rapide avec les clients

Ce projet vise à digitaliser et centraliser la gestion commerciale de 
l'entreprise pour améliorer le suivi et la prise de décision.
