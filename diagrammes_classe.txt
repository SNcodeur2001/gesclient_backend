Ah je vois — tu veux des relations simples avec juste les multiplicités (0..1, 1, 0..*) sans les symboles de composition/agrégation. Voici :

```plantuml
@startuml
skinparam linetype ortho
skinparam nodesep 40
skinparam ranksep 60
skinparam classAttributeIconSize 0

' =================== ENUMS ===================

enum Role {
  DIRECTEUR
  COMMERCIAL
  COLLECTEUR
}

enum ClientType {
  APPORTEUR
  ACHETEUR
}

enum ClientStatut {
  ACTIF
  PROSPECT
  INACTIF
}

enum CommandeType {
  SUR_PLACE
  A_DISTANCE
}

enum CommandeStatut {
  EN_ATTENTE_ACOMPTE
  EN_PREPARATION
  PRETE
  FINALISEE
}

enum PaiementType {
  ACOMPTE
  SOLDE
}

enum ModePaiement {
  ESPECES
  VIREMENT
  CHEQUE
  MOBILE_MONEY
}

enum FactureType {
  PROFORMA
  DEFINITIVE
}

enum FactureStatut {
  GENEREE
  ENVOYEE
  TELECHARGE
}

enum NotificationType {
  NOUVELLE_COLLECTE
  ACOMPTE_RECU
  COMMANDE_PRETE
  COMMANDE_FINALISEE
  COMMANDE_EN_ATTENTE
}

' =================== CLASSES ===================

class User {
  -id: String
  -nom: String
  -prenom: String
  -email: String
  -password: String
  -role: Role
  -actif: Boolean
  -createdAt: DateTime
}

class Client {
  -id: String
  -nom: String
  -prenom: String
  -email: String
  -telephone: String
  -adresse: String
  -type: ClientType
  -statut: ClientStatut
  -totalRevenue: Float
  -notes: String
  -assignedUserId: String
  -createdAt: DateTime
  -deletedAt: DateTime
}

class Commande {
  -id: String
  -reference: String
  -type: CommandeType
  -statut: CommandeStatut
  -montantHT: Float
  -tva: Float
  -montantTTC: Float
  -acompteMinimum: Float
  -acompteVerse: Float
  -soldeRestant: Float
  -createdAt: DateTime
}

class CommandeItem {
  -id: String
  -produit: String
  -quantite: Float
  -prixUnitaire: Float
}

class Collecte {
  -id: String
  -montantTotal: Float
  -notes: String
  -createdAt: DateTime
}

class CollecteItem {
  -id: String
  -typePlastique: String
  -quantiteKg: Float
  -prixUnitaire: Float
}

class Paiement {
  -id: String
  -type: PaiementType
  -montant: Float
  -modePaiement: ModePaiement
  -createdAt: DateTime
}

class Facture {
  -id: String
  -numero: String
  -type: FactureType
  -statut: FactureStatut
  -montantHT: Float
  -tva: Float
  -montantTTC: Float
  -envoyeeWhatsApp: Boolean
  -downloadToken: String
  -createdAt: DateTime
}

class Notification {
  -id: String
  -type: NotificationType
  -message: String
  -lu: Boolean
  -lien: String
  -createdAt: DateTime
}

class AuditLog {
  -id: String
  -action: String
  -entite: String
  -entiteId: String
  -ancienneValeur: JSON
  -nouvelleValeur: JSON
  -createdAt: DateTime
}

class RefreshToken {
  -id: String
  -tokenHash: String
  -expiresAt: DateTime
  -revokedAt: DateTime
}

' =================== RELATIONS ENUMS ===================

User ..> Role
Client ..> ClientType
Client ..> ClientStatut
Commande ..> CommandeType
Commande ..> CommandeStatut
Paiement ..> PaiementType
Paiement ..> ModePaiement
Facture ..> FactureType
Facture ..> FactureStatut
Notification ..> NotificationType

' =================== RELATIONS CLASSES ===================

User "1" -- "0..*" Client : assigne
User "1" -- "0..*" Commande : commercial
User "1" -- "0..*" Collecte : collecteur
User "1" -- "0..*" Notification : reçoit
User "1" -- "0..*" AuditLog : génère
User "1" -- "0..*" RefreshToken : possède
User "1" -- "0..*" Paiement : valide

Client "1" -- "0..*" Commande : acheteur
Client "1" -- "0..*" Collecte : apporteur

Commande "1" -- "1..*" CommandeItem : contient
Commande "1" -- "0..*" Paiement : reçoit
Commande "1" -- "0..*" Facture : génère
Commande "1" -- "0..*" Notification : déclenche

Collecte "1" -- "1..*" CollecteItem : contient

@enduml
```

---

Relations simples avec juste `--` et les multiplicités, exactement comme dans ton exemple. Plus lisible et suffisant pour un mémoire de fin de cycle.
